# Bargain Hunter

## 📋 프로젝트 소개

**국내 관광 지원 서비스** - 전국의 관광지, 문화명소, 투어, 레저 장소 등을 지도로 탐색하고, 리뷰를 남기며, 채팅형 가격 비교까지 가능한 여행 탐색 플랫폼

### 주요 기능

- 🗺️ **지도 탐색**: 행사/관광지/가게 정보 제공 (Tour API 활용)
- 📝 **리뷰**: 경험 기반 리뷰 작성 및 조회
- 💬 **채팅형 가격 비교**: LLM 기반 챗봇으로 지역별 상품 가격 문의

## 🏗️ 시스템 아키텍처

### MSA 기반 서비스 구조

```
├─ gateway/      # API Gateway (Spring Cloud Gateway)
├─ auth/         # 인증 및 사용자 관리 서비스
├─ review/       # 리뷰 서비스 (Kafka 메시징)
├─ tour/         # 관광지 정보 서비스 (TourAPI 연동)
└─ util/         # LLM 서비스 (Gemini API)
```

### 주요 아키텍처 특징

1. **MSA (Microservices Architecture)**

   - 각 도메인별 독립적인 서비스 구성
   - 서비스 간 느슨한 결합
   - 독립적인 배포 및 확장 가능

2. **API Gateway 패턴**

   - 모든 요청의 단일 진입점
   - JWT 기반 중앙 인증 처리
   - 서비스 라우팅 및 로드밸런싱

3. **이벤트 기반 아키텍처**

   - Spring Event + @Async 활용
   - 비동기 이메일 발송 처리
   - API 응답 속도 92% 개선

4. **Kubernetes 기반 배포**
   - Docker 컨테이너화
   - 자동 배포 파이프라인 (GitHub Actions)
   - PR Merge 시 자동 빌드 및 배포

## 🔑 핵심 성과

### 성능 개선

- 이메일 발송 비동기 처리 → **API 응답 속도 92% 개선** (2.5초 → 0.2초)
- Redis Hash + HINCRBY를 활용한 동시성 제어

### 보안 강화

- PKCE 적용으로 **OAuth2 Authorization Code Interception** 방지
- Refresh Token DB 저장 → 토큰 탈취 시 즉시 대응 가능
- Gateway 단일 JWT 검증으로 인증 오버헤드 제거

### 아키텍처

- MSA 기반 독립적인 서비스 구조
- 이벤트 기반 비동기 처리로 시스템 응답성 향상
- Kubernetes를 활용한 자동 배포 및 스케일링

## 📚 배운 점

### MSA 설계 경험

- 서비스 간 통신 및 데이터 일관성 관리
- Gateway를 통한 인증 정보 전달 방법

- 팀 간 기술 스택 통일 및 코드 컨벤션 정립
