# II. 기술 스택

## 1️⃣ Frontend

| 기술             | 선택 이유                                                    |
| ---------------- | ------------------------------------------------------------ |
| **React**        | 컴포넌트 기반 구조로 복잡한 관리자 화면을 효율적으로 관리    |
| **Vite**         | 빠른 개발 서버와 번들링 속도로 생산성 향상                   |
| **TypeScript**   | 대규모 상태 및 API 모델 관리 시 타입 안정성 확보             |
| **Tailwind CSS** | 관리자 페이지 특성상 빠른 UI 구성 및 일관된 스타일 적용      |
| **Zustand**      | 전역 상태를 단순하게 관리하기 위한 경량 상태 관리 라이브러리 |
| **React Query**  | 서버 상태와 캐시 관리, 로딩/에러 처리 단순화                 |

---

## 2️⃣ Backend

| 기술                  | 선택 이유                                           |
| --------------------- | --------------------------------------------------- |
| **Java 17**           | 최신 LTS 버전 기반 안정적인 서버 개발               |
| **Spring Boot 3.x**   | 생산성과 유지보수성을 고려한 표준 백엔드 프레임워크 |
| **Spring Security**   | 인증/인가 정책의 일관된 적용                        |
| **JPA (Hibernate)**   | 단순 CRUD 및 도메인 중심 개발에 적합                |
| **MyBatis**           | 복잡한 검색 조건 및 성능 튜닝이 필요한 쿼리 처리    |
| **MySQL**             | 관계형 데이터 구조에 적합한 안정적인 RDBMS          |
| **Redis**             | 세션 관리 및 캐시 처리                              |
| **Swagger (OpenAPI)** | API 명세 자동화 및 협업 효율 향상                   |

---

## 3️⃣ Infra / DevOps

| 기술        | 선택 이유                         |
| ----------- | --------------------------------- |
| **Docker**  | 환경 일관성 확보 및 배포 단순화   |
| **NginX**   | 정적 리소스 서빙 및 리버스 프록시 |
| **AWS EC2** | 유연한 인프라 구성                |
| **GitLab**  | 코드 관리 및 협업                 |
| **Nexus**   | 폐쇄망 환경에서의 패키지 관리     |

## 5️⃣ 기술 선택 전략 요약

### 🧩 하이브리드 ORM 전략

- **JPA**: 단순 CRUD 및 도메인 중심 로직
- **MyBatis**: 복잡한 동적 검색, 성능 튜닝이 필요한 영역

> 기술을 일괄 적용하지 않고 **업무 특성에 맞게 선택**하여  
> 개발 생산성과 성능을 동시에 확보했다는 점이 핵심입니다.
