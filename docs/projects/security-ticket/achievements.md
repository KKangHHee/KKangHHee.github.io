# V. 주요 성과 및 배운 점

## 🎯 주요 성과

### 1️⃣ 복잡한 검색 쿼리 성능 개선

- JPA 기반 동적 쿼리의 한계를 인지하고 **MyBatis로 전환**
- 검색 조건이 많아질수록 유지보수가 어려웠던 구조 개선
- 인덱스 설계와 쿼리 구조 최적화를 통해 조회 성능 향상

**결과**

- 검색 응답 속도 개선
- 조건 추가 시 코드 수정 범위 감소
- SQL 가독성 및 유지보수성 향상

---

### 2️⃣ 관리자 중심 백오피스 구조 설계

- 관리자 사용 흐름을 기준으로 기능 정의
- 상태 기반 워크플로우 설계 (`OPEN → IN_PROGRESS → DONE`)
- 페이징 / 정렬을 표준화하여 대량 데이터 환경 대응

**결과**

- 실무 환경과 유사한 관리자 시스템 구현
- 반복 작업 최소화 및 운영 효율 향상

---

### 3️⃣ 안정적인 인증·보안 구조 구축

- Spring Security + JWT 기반 인증 구조 적용
- Stateless 방식으로 서버 부하 감소
- 관리자 전용 API 접근 제어

**결과**

- 보안 요구사항 충족
- 확장 가능한 인증 구조 확보

---

## 📘 배운 점

### 1️⃣ 기술 선택은 “문제 해결” 중심이어야 한다

- JPA가 항상 최선은 아니라는 점을 경험
- **문제 성격에 맞는 기술 선택의 중요성** 체감
- 복잡한 조회 로직에서는 SQL 중심 접근이 더 효율적임을 학습

---

### 2️⃣ 성능 문제는 구조에서 시작된다

- 단순한 쿼리 튜닝보다
  - 도메인 설계
  - 조회 책임 분리
  - 적절한 계층화
- 이러한 구조적 접근이 성능 개선의 핵심임을 인지

---

### 3️⃣ “동작하는 코드”와 “운영 가능한 코드”는 다르다

- 예외 처리, 인증, 로그, 확장성을 고려해야
  실제 서비스에 가까운 코드가 된다는 점을 체감
- 관리자 서비스 특성상 **안정성과 가독성**이 중요함을 학습

---

## ✅ 프로젝트를 통해 성장한 점

- 백엔드 전반 구조를 스스로 설계하고 개선하는 경험
- 성능 병목 지점을 분석하고 기술적으로 해결한 경험
- 실무 기준에 가까운 관리자 서비스 구현 역량 강화
