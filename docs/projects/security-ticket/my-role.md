# IV. 담당 역할 및 기여도

## 🔹 프로젝트 내 역할

본 프로젝트에서 **백엔드 개발을 중심으로 한 핵심 기능 구현**을 담당하였다.  
기획 의도에 맞는 구조 설계부터 성능 개선, 운영 관점의 안정성 확보까지 전반적인 개발을 주도하였다.

- 백엔드 API 설계 및 구현
- 데이터베이스 모델링
- 관리자 백오피스 핵심 기능 개발
- 성능 개선 및 쿼리 최적화

---

## 1️⃣ 백엔드 아키텍처 설계

### 📌 기여 내용

- Spring Boot 기반 백엔드 구조 설계
- 도메인 중심 패키지 구조 적용
- 공통 응답 및 예외 처리 구조 설계

### 🎯 성과

- 기능 확장 시 코드 변경 범위 최소화
- 유지보수성과 가독성 향상

---

## 2️⃣ 복잡한 검색 로직 성능 개선

### 📌 문제 상황

- JPA 기반 동적 검색 로직 사용 시
  - 조건 분기 증가
  - 쿼리 가독성 저하
  - 성능 저하 발생

### 🛠 해결 방법

- **MyBatis 도입**
  - 동적 SQL 기반 검색 쿼리 분리
  - 복잡한 조건을 SQL 레벨에서 처리
- 인덱스 컬럼 재설계

### 🎯 성과

- 검색 쿼리 응답 속도 개선
- SQL 가독성 및 유지보수성 향상
- 검색 조건 추가 시 유연한 대응 가능

---

## 3️⃣ 관리자 기능 중심의 API 설계

### 📌 기여 내용

- 관리자 사용 시나리오 기반 API 설계
- 상태 기반 티켓 워크플로우 구현
- 페이징 / 정렬 처리 표준화

### 🎯 성과

- 관리자 업무 흐름에 맞춘 기능 제공
- 대량 데이터 환경에서도 안정적인 조회 성능 확보

---

## 4️⃣ 보안 및 인증 처리

### 📌 기여 내용

- Spring Security 기반 인증/인가 구현
- JWT를 활용한 Stateless 인증 구조 적용
- 관리자 전용 API 접근 제어

### 🎯 성과

- 불필요한 세션 의존 제거
- API 보안 수준 강화

---

## 5️⃣ 협업 및 문제 해결 방식

### 📌 접근 방식

- 기능 구현 전 요구사항을 명확히 정리
- 성능 이슈 발생 시 **원인 분석 → 개선 → 검증** 순으로 접근
- 단기 구현보다 **확장성과 유지보수성** 우선 고려

### 🎯 결과

- 기능 추가 및 수정 시 구조 변경 최소화
- 실무 환경에 가까운 백엔드 개발 경험 축적

---

## ✅ 담당 역할 요약

- 백엔드 핵심 기능 전반 구현
- 성능 병목 구간 직접 분석 및 개선
- 관리자 서비스 특성에 맞는 구조 설계
- 실무 기준의 보안·운영 관점 반영
