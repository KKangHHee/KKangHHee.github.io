# I. 프로젝트 개요

## 1️⃣ 목적 / 배경

### 🎯 프로젝트 배경

SK플래닛 T아카데미 웹 풀스택 과정 7기 최종 프로젝트로,  
정보보호 및 컴플라이언스 점검 프로세스의 **비효율적인 운영 방식**을 개선하는 것을 목표로 진행했습니다.

### 🔍 기존 문제 인식

기존 정보보호 컴플라이언스(ISMS-P, ISO27001) 점검은  
**Excel 문서와 이메일 중심의 수동적인 방식**으로 운영되고 있었습니다.

이로 인해 다음과 같은 문제가 발생했습니다:

- 점검 이력 관리 및 추적의 어려움
- 진행 상태에 대한 실시간 파악 불가
- 역할(관리자/고객사) 기반 접근 제어 미흡
- 반복적인 커뮤니케이션으로 인한 업무 비효율

---

## 2️⃣ 해결 목표

> **정보보호 컴플라이언스 점검 프로세스의 디지털 전환**

- 검증 프로세스의 웹 기반 표준화
- 역할 기반 권한 관리 및 접근 제어
- 점검 상태 시각화 및 자동 알림 제공
- 반복 업무 최소화를 위한 템플릿 기반 검증 흐름 설계

---

## 3️⃣ 프로젝트 기간

**2025.04 ~ 2025.05 (2개월)**

---

## 4️⃣ 팀 구성

| 구분       | 인원 | 주요 역할                              |
| ---------- | ---- | -------------------------------------- |
| **BE-A팀** | 4명  | 기획, 인프라, 사용자 관리, 비동기 알림 |
| **BE-B팀** | 4명  | 컴플라이언스 검증/관리, 보안 티켓 관리 |
| **FE팀**   | 3명  | 프론트엔드 개발                        |

**개인 역할**: Backend Developer (B팀)

- 보안 티켓 관리 및 컴플라이언스 검증 기능 담당
- 세션 기반 인증/인가 시스템 설계
- 사용자 관리 CRUD 및 복합 검색 API 구현

---

## 5️⃣ 주요 기능 요약

### 🔐 인증 / 인가

- 최고 관리자를 통한 계정 발급 및 Role 관리
- 세션 기반 로그인 상태 관리
- 역할 기반 접근 제어

### 👥 관리자 / 고객사 회원 관리

- 사용자 CRUD 및 페이지네이션
- 상태 및 기간 기반 필터링 검색

### 📋 컴플라이언스 신청 / 검증

- 고객사의 보안 점검 요청 접수
- 관리자 측 표준 문항 기반 검증 진행
- 진행 상태 관리 및 일정 알림 제공

---

## 🎯 프로젝트 목표 달성도

### ✅ 달성한 목표

- 웹 기반 검증 프로세스 표준화 완료
- 역할 기반 권한 관리 시스템 구축
- 복합 검색 기능의 성능 최적화 (평균 응답시간 36% 개선)
- 팀 간 API 표준화로 협업 효율성 향상

### 📈 향후 개선 방향

- 실시간 알림 기능 고도화
- 검증 이력 통계 대시보드 구축
- 외부 시스템 연동 API 개발
